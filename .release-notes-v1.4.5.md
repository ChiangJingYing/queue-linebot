## What's Changed

自 `v1.4.4` 以來，這個版本主要補上了跨平台 Telegram admin 推播，讓管理員可以在 Telegram 端追蹤 LINE / Discord 的關鍵排隊事件，並同步補齊設定說明文件。

### Cross-platform Telegram admin notifications
- 新增 / 擴充 Telegram admin notification function，通知訊息現在可夾帶平台標記，例如：`平台：Line`、`平台：Discord`、`平台：Telegram`。
- 沿用既有 `TelegramAdminNotificationService`，不額外拆新 service，維持原本 admin 通知偏好與分類機制。
- 保留 `TELEGRAM_NOTIFICATION_CATEGORIES` 模型，管理員仍可透過既有的 `/admin/notify ...` 與通知設定 UI 進行開關。

### LINE admin operations now broadcast to Telegram admins
- LINE admin 成功操作後，會補送 Telegram admin 推播。
- 已接入的 LINE admin 事件包括：
  - `/admin/serve`
  - `/admin/clear`
  - `/admin/vip toggle [on/off]`
  - `/admin/vip clear`
- 其中：
  - 叫號相關通知使用 `serve`
  - 其他管理操作通知使用 `admin_action`

### LINE / Discord user queue events now broadcast to Telegram admins
- 一般使用者從 LINE / Discord 加入隊列時，會送出 Telegram admin `join` 推播。
- 一般使用者從 LINE / Discord 確實放棄排隊時，會送出 Telegram admin `cancel` 推播。
- 目前對應如下：
  - `join`：Telegram / Line / Discord
  - `cancel`：Telegram / Line / Discord
  - `serve`：Telegram / Line
  - `admin_action`：Telegram / Line
  - `skip`：Telegram
  - `register`：Telegram
  - `error`：Telegram

### Runtime wiring fixes
- 修正 `main.py` 中的 runtime wiring，確保：
  - `LineBotHandler` 正確注入 `telegram_sender`
  - `DiscordCommandService` 正確注入 `telegram_sender`
- 這個修正讓 LINE / Discord 端新增的 Telegram admin 推播在實際容器執行時能正常送出，而不只是測試通過。

### README improvements
- 補充 README 中「推播通知」相關說明。
- 明確列出每個 `/admin/notify` 開關代表的事件類型。
- 補充每個通知類別目前支援的來源平台。
- 說明目前 `/admin/notify` 是 Telegram admin 的通知接收偏好設定，而非 LINE / Discord 各自獨立的通知設定介面。
