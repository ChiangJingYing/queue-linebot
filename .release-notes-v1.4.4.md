## What's Changed

自 `v1.4.3` 以來，這個版本主要補強了 Discord interaction 相容性、Telegram 叫號推播，以及 README 文件整理。

### Discord interaction 修復
- 修正 Discord component schema：button `style` 改為回傳 Discord 要求的數字值，而不是字串樣式名稱。
- 修正註冊流程選項過多時的 row limit 問題：每個 action row 最多 5 顆 button，超過時自動分列。
- 保留既有 interaction 相容行為，包含 `type == 1 / 2 / 3 / 5` 與既有 command option 處理。
- 補上對應 regression tests，避免 `/menu`、`/help`、`/register` 再出現 interaction schema 錯誤。

### Telegram 推播修復
- 修正 Telegram 使用者被叫號時收不到推播的正式鏈路。
- `Notifier` 現在會依平台分流通知：Discord user 走 Discord DM、Telegram user 走 Telegram sender，其餘維持 LINE push。
- Telegram webhook / command 入口會標記 `telegram_user:<user_id>`，讓後續叫號與 skip 通知能正確送往 Telegram。
- 補上 Telegram 叫號與 skip 通知測試，確認 user 通知與 admin 通知都能正確送達。

### 架構與文件
- README 改為中文主體，保留必要的中英混用術語（如 `Discord interactions`、`Discord DM`、`Dashboard`）。
- 新增三平台功能對照表與設計說明，整理 shared flow、platform adapter / presenter 分工，以及未來修改原則。
- 明確補上平台推播與管理通知差異，降低後續維護誤解。
